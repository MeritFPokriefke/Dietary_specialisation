File affiliation to manuscript components is denoted using labels (e.g. `#Variation in general diet`), allowing scripts to be directly linked to the corresponding assumptions and predictions. The repository is organised into three main components: code, data, and output.

The `Code/` directory contains all scripts used for analyses, including models (M; with associated STAN files), plots (P), and tables (T). Scripts are numerically ordered according to the key assumptions and predictions they address (e.g. 01, 02, 03). Where multiple scripts relate to the same component, additional numbering is used (e.g. M1, M2).

Scripts used for metabarcoding data processing are located within the `Data/` directory.


## Data

### Data read abundances

**File:** `birds2_280825.csv`\
**Source:** bird diet dataset

Each row represents one sampling event (individual × time × replicate). Variables starting with `RRA_` represent relative read abundance for a given taxonomic group.

#### Columns (in data frame order)
RRA_Lepidoptera: relative read abundance of Lepidoptera reads (order level)\
RRA\_\*\*\*: as follows in data frame
TreatmentB: density treatment blue tit\
TreatmentG: density treatment great tit\
year: year when measurement was taken\
Ha: size of the subplot in hectar\
Doublebox: nest box in close proximity (yes = 1, 0 = no)
Type: bird type (AF = adult female, AM = adult male, chick 1 = nestling day 10, chick 2 = nestling day 15)\
Species: bird species (0 = blue tit, 1 = great tit)\
AdateSample: date measurement was taken in April days (first day of April = 1)\
BroodID: unique identifier for each brood\
WithinNestRep: repeated measurement within a brood, BroodID + nestlings type (chick1 = nestling day 10, chick2 = nestling day 15)\
SPY: identifier for subplot + year Subplot: identifier for subplot\
RingNumber: unique identifier adult bird

## How to run

Run scripts in order as presented in the directory `Code/Models/`. For metabarcoding scripts in `Data/COI140725`, run one (birds or environment) then close all and run the other.

## Outputs

Model outputs are stored in the `Output/Models/` directory and are named according to their corresponding script identifier and creation date. Table outputs are formatted for inclusion in LaTeX. Plot outputs are saved as PDF files and, if required, further refined in Affinity Designer (Version 2.2.1) without modification of the underlying data.

## Notes

Running time of models varies between 2 min and ~14 hours depending on the complexity (e.g. loop). Used seeds are noted in the STAN model quantification.
